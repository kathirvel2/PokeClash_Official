Telegram Pokémon Battle System UI Specification1. IntroductionThis document provides a detailed specification for a turn-based Pokémon battle user interface (UI) designed for a Telegram bot. The system is built around a single, dynamically updated message, providing a seamless experience for the player. The goal is to present all relevant information clearly and concisely, allowing for strategic decision-making.2. Core UI ComponentsThe entire battle interface is contained within a single Telegram message, which is composed of three distinct parts:Dynamic Image: A generated image showing the two battling Pokémon.Caption: The text content of the message, containing all gameplay information.Inline Keyboard: A set of buttons below the message for user input.Text Diagram: Overall Message Structure+-----------------------------------------------------+

3. Detailed Caption StructureThe caption is the core of the UI. It is a single block of text formatted using monospace fonts for alignment and HTML for styling (<b>, <code>). It is divided into four main sections, ordered from top to bottom.Text Diagram: Caption Layout Template{ACTION_LOG_SECTION}

{OPPONENT_STATUS_SECTION}

{PLAYER_STATUS_SECTION}

{CURRENT_PLAYER_MOVE_DETAILS_SECTION}
3.1. Action Log SectionPurpose: To narrate the events of the turn (e.g., which moves were used, their effectiveness, and damage dealt).State: This section is updated after an action is taken.3.2. Opponent & Player Status SectionsPurpose: To display the current state of each Pokémon in the battle.Content: Includes Name, Types, Level, HP (current/max), a visual health bar, and a Current turn: indicator. The indicator is the primary visual cue for whose move it is.Text Diagram: Status Block StructureOpponent's {POKEMON_NAME} [{TYPE_1} / {TYPE_2}]

3.3. Current Player's Move Details SectionPurpose: To provide the active player with all information needed to make a strategic decision.Content: Lists the four moves of the Pokémon belonging to the player whose turn it currently is.4. Inline Keyboard StructureThe keyboard provides the interface for player actions. It remains visible throughout the battle and is updated at the start of each player's turn to reflect their available moves.Text Diagram: Keyboard Layout+------------------+------------------+
